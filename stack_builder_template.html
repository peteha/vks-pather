VCF - VKS Stack Builder

Interactive compatibility windows for a VCF stack running vSphere Kubernetes Service (VKS)

What this tool does

Kubernetes ships a new minor version roughly every four months, and every layer of the stack — vCenter, ESX, NSX, the vSphere Supervisor, VKS, the workload-cluster Kubernetes releases (TKr) and their add-on packages — supports only a window of the other layers' versions. This tool makes that moving window visible: pin a version on any layer and the green bands show how far every other layer can range while staying supported; add more pins to narrow the stack down to a concrete, fully supported combination. A supported solution requires every layer to keep at least one compatible version, so any version that would leave another layer with no options is dimmed and cannot be selected.

Where the data comes from: every “Compatible / Incompatible / Not Supported” verdict is a cell from the official Broadcom Product Interoperability Matrix, exported 16 Jul 2026. Where the matrix has no direct rows for a component pair, the tool shows a clearly marked derived window inferred through an intermediate component — labelled inference, not a matrix cell. The matrix is a point-in-time snapshot: re-export it and re-verify before relying on a combination. Independent analysis — not Broadcom guidance; not affiliated with Broadcom Inc.
